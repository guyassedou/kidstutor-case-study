# Public-Safe Media Shot List

No media should be captured from an account containing real child or parent data.

## Recommended screenshots

1. Synthetic child home screen
2. Grade-selection or practice setup
3. Representative question with no identifying information
4. Touch scratchpad interaction
5. Worked feedback after an intentionally synthetic answer
6. Progress or achievement view using demo values
7. Personalization or shop view with synthetic ownership state
8. Sanitized parent-report example generated from synthetic data

## Two-minute walkthrough

Suggested sequence:

1. Introduce the learning problem and target experience.
2. Start a synthetic practice session.
3. Show question legibility and Hebrew vocalization.
4. Demonstrate scratch work on a touch-sized viewport.
5. Submit an incorrect answer and show recovery-oriented feedback.
6. Complete a short synthetic session.
7. Show progress and personalization.
8. Close with the product operating model: feedback, scoped decisions, AI-assisted implementation, and verification.

## Capture rules

- Use a dedicated demo account.
- Use synthetic names and progress.
- Hide browser bookmarks, notifications, and unrelated tabs.
- Do not open developer tools during the recording.
- Do not display production URLs, database identifiers, or admin functions.
- Review the finished video frame by frame before publishing.
- Strip unnecessary metadata from exported media.
