# Privacy and Disclosure Boundary

## Public materials

This repository may include:

- High-level product descriptions
- Independently written decision narratives
- Simplified architecture diagrams
- Dated build and test evidence
- Sanitized screenshots or recordings
- Synthetic demonstration data

## Excluded materials

This repository must not include:

- Child or parent names
- Account identifiers
- Progress or usage records
- Raw feedback
- Production screenshots containing personal information
- Production database details
- Source code or private Git history
- Authentication or authorization implementation
- Production configuration
- Internal prompts or complete agent instructions
- Proprietary curriculum and algorithms
- Commercial roadmap

## Synthetic data requirements

Any future demonstration data must:

- Be created specifically for the demonstration
- Use fictional people and accounts
- Avoid distributions copied from production
- Be visually labeled as demo or synthetic data
- Remain isolated from production systems
- Be resettable without affecting real users

## Media review

Before publication, inspect every screenshot and every frame of video for:

- Names and avatars
- Browser tabs and bookmarks
- URLs and account identifiers
- Notifications
- Developer tools
- Local paths
- Production metrics
- Image metadata
- Hidden content visible during transitions

## Claims

Technical claims should be tied to a dated repository snapshot. Educational, behavioral, or commercial outcome claims require separate evidence and review.

## Product status

KidsTutor is a proprietary product under development. This repository is a portfolio case study, not an open-source distribution, public dataset, or production system.
